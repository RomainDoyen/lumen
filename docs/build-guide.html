Documentation du projet

Lumen est une application de compression d’images, de préparation de packs WordPress (tailles natives, WebP + JPEG, métadonnées) et de génération de packs d’icônes. Elle s’utilise en application Electron ou en ouvrant les pages dans un navigateur ; le traitement image reste local sur la machine de l’utilisateur.

Présentation

L’outil permet de réduire la taille de fichiers image (PNG, JPEG, AVIF vers WebP), de produire un ZIP prêt pour WordPress (plusieurs images, dossier par slug, metadata.json, blocs Gutenberg) et de générer des kits d’icônes PNG. Le rendu et l’export d’images s’exécutent dans le navigateur embarqué (Electron) ou dans votre navigateur ; seule la suggestion de textes via Mistral (optionnelle) envoie une miniature vers l’API Mistral avec votre propre clé.

En bureau, Electron charge des styles dédiés et contourne la protection par domaine. En navigateur, les mêmes pages HTML peuvent être servies en statique ; la licence et config.env s’appliquent selon ton déploiement.

Modes d’exécution

Mode Comment l’ouvrir Particularités
Application Lumen npm start ou installateur dans release/ Fenêtre Electron, classe cc-desktop, pas d’analytics Google, menu macOS minimal, icône système
Lumen (navigateur) Servir le dossier app/ (entrée app/pages/index.html) ou déployer en statique Même interface que le bureau sans couche Electron ; protection licence / domaine selon config.env si déployé en ligne

Fonctionnalités

Compression WebP

Import multiple PNG / JPEG / AVIF. Choix du format de sortie (auto, carré, 16:9, 9:16). Profils de compression et curseur de qualité.

Prévisualisation

Comparaison avant / après, dimensions, ratio de compression et stats agrégées.

Export

Téléchargement fichier par fichier, tout télécharger, ou archive ZIP avec fichier README de rapport.

Pack WordPress

Plusieurs images → 10 fichiers chacune (5 tailles × WebP + JPEG), metadata.json avec media[], snippets <picture> par slug (wordpress.html).

Métadonnées & Mistral

Saisie manuelle ou suggestion IA (clé utilisateur). Limite API : message affiché, saisie toujours possible. Brouillon auto (IndexedDB).

Kit d’icônes

À partir d’une image source : PNG aux tailles 16 à 512 px, téléchargement unitaire ou ZIP (icon.html).

Confidentialité et sécurité

  • Les images sont lues en mémoire dans le navigateur ; elles ne sont pas téléchargées vers nos serveurs applicatifs pour traitement.
  • Pack WordPress : les fichiers du pack (redimensionnement, WebP, JPEG) restent locaux. Si tu actives Mistral, seule une miniature JPEG (max. 1024 px) par image est envoyée à api.mistral.ai avec ta clé (stockée dans le navigateur, jamais chez Lumen).
  • Le brouillon du pack WordPress est enregistré dans IndexedDB sur ta machine (reprise après changement d’onglet Lumen ou rechargement de la page).
  • Sur le site web, des scripts tiers peuvent être présents selon la configuration (par ex. mesure d’audience) ; ils sont désactivés dans l’app Electron Lumen.
  • La communication réseau hors navigateur (mises à jour Electron, polices Google si chargées) dépend de votre environnement.

Prérequis

Pour développer ou packager

  • Node.js LTS (18 ou 20+) et npm
  • Environ 500 Mo d’espace pour Electron et les dépendances
  • Premier lancement : téléchargement du binaire Electron (réseau)

Pour l’utilisateur final

  • Application : aucun Node requis ; installer l’AppImage, le .deb, le .exe ou ouvrir le .dmg selon la plateforme.
  • Web : navigateur récent avec JavaScript activé.

Build Windows depuis Linux

Les installeurs NSIS demandent souvent Wine (et parfois Mono) si tu compiles depuis Linux. Sinon, exécute npm run dist sur Windows.

Installation (développeur)

À la racine du projet :

cd lumen
npm install

Icônes du paquet

npm run icons

Produit build/icon.png (macOS / Linux) et build/icon.ico (Windows, multi-tailles 16–256 px) à partir du SVG Lumen. Obligatoire avant npm run dist, surtout sur Windows — sans icon.ico, l’exécutable affiche l’icône Electron par défaut.

Utilisation — Compression WebP

  1. Ouvre app/pages/index.html (ou lance Lumen) puis reste sur Images.
  2. Dépose des fichiers ou clique la zone / « parcourez vos fichiers » pour sélectionner PNG, JPEG ou AVIF.
  3. Choisis le format de cadrage (auto, carré, paysage, portrait), le niveau de compression et la qualité WebP.
  4. Clique sur Compresser et convertir en WebP.
  5. Récupère les fichiers individuellement, tout en une fois, ou via Télécharger ZIP (archive + README de stats).
  6. Recommencer réinitialise la session sans recharger la page.

Utilisation — Pack WordPress

Module pensé pour gagner du temps sur un site WordPress : une ou plusieurs images sources produisent un ZIP unique avec tout le nécessaire pour l’import média et le collage Gutenberg.

Parcours type

  1. Ouvre app/pages/wordpress.html ou l’onglet Pack WordPress dans Lumen.
  2. Glisse-dépose une ou plusieurs images (PNG, JPEG, AVIF, WebP) ou utilise + Ajouter pour compléter le lot.
  3. Renseigne l’URL du site (optionnel) : utilisée dans les chemins des snippets ; sinon le placeholder {{SITE_URL}} est conservé.
  4. Pour chaque image, complète les métadonnées (slug, titre, 3 variantes d’alt, légende, description). Un préremplissage depuis le nom de fichier est proposé à l’import.
  5. Optionnel — Saisis ta clé API Mistral (conservée en localStorage sur ce navigateur) puis Suggérer toutes les images avec Mistral ou le bouton Mistral sur une carte. Vérifie toujours les textes avant export.
  6. Clique sur Générer le pack WordPress (ZIP), puis Télécharger le ZIP.

Contenu du ZIP

Élément Description
{slug}/ 10 fichiers par image : original, large (1024), medium_large (768), medium (300), thumbnail (150) — chacun en .webp et .jpg. EXIF retirés au ré-encodage. La miniature WP est recadrée en carré centré.
metadata.json Tableau media[] : champs REST (alt_text, caption, etc.), liste des fichiers, JSON-LD, référence Gutenberg.
gutenberg/{slug}.html Bloc <!-- wp:html --> avec balise <picture> (srcset WebP + fallback JPEG).
README.txt Rappel de la structure et des étapes d’import.

Brouillon automatique

Tant que tu n’as pas cliqué sur Tout effacer, le travail en cours (images + métadonnées + URL du site) est sauvegardé dans IndexedDB. En revenant sur Pack WordPress après un autre onglet Lumen (Images, Icônes) ou un rechargement, un message Brouillon restauré réaffiche le lot.

Mistral — limites et saisie manuelle

  • Modèle vision utilisé : ministral-14b-latest (appel direct depuis le navigateur).
  • Si la limite API est atteinte (HTTP 429 ou message équivalent), une bannière s’affiche : tu peux continuer à taper ou corriger tous les champs à la main.
  • Lors d’une suggestion sur plusieurs images, les images déjà traitées conservent leurs textes ; le reste reste en saisie manuelle.

Import sur WordPress

L’upload REST automatique depuis Lumen n’est pas intégré (CORS, authentification). Workflow recommandé :

  1. Importer les fichiers via Médias ou FTP dans wp-content/uploads/ (structure libre ; les chemins du ZIP suivent lumen/{slug}/ si tu respectes l’URL renseignée).
  2. Coller le HTML depuis gutenberg/{slug}.html dans un bloc HTML personnalisé.
  3. Mettre à jour alt_text, légende et description via l’admin ou PATCH /wp/v2/media/{id} en t’aidant de metadata.json.
Slug unique : si deux fichiers produisent le même slug, Lumen ajoute un suffixe (-2, -3…). Pense à les renommer avant export si tu veux des noms précis.

Utilisation — Générateur d’icônes

  1. Ouvre app/pages/icon.html ou l’onglet Icônes app dans Lumen.
  2. Importe une image PNG, JPEG ou SVG (glisser-déposer ou parcourir).
  3. Vérifie la prévisualisation source puis Générer toutes les icônes.
  4. Télécharge chaque taille ou l’archive ZIP regroupant les PNG 16×16 à 512×512.
  5. Recommencer permet de changer de fichier source.

Note : ce module produit des PNG ; la compilation en fichier .ico multi-résolution peut nécessiter un outil externe si tu en as besoin pour une ancienne version de Windows.

Configuration (config.env)

Optionnel : fichier à la racine du projet, chargé par env-loader.js via une requête HTTP en mode web. Exemples de clés documentées dans le code :

  • LICENSE_KEY, AUTHORIZED_DOMAINS — limitation du déploiement web avec license-protection.js
  • PROTECTION_ENABLED, DEBUG_MODE
  • APP_VERSION, APP_AUTHOR

En application Lumen, la protection domaine est contournée : l’objectif est un usage hors ligne / fichier local sans bloquer l’interface.

Si config.env est absent, des valeurs par défaut sont utilisées (voir env-loader.js).

Toutes plateformes et CI

Linux

npm run dist sur Linux → paquets Linux. L’AppImage est pratique pour un binaire portable.

Windows

Build sur Windows pour éviter Wine, ou chaîne CI windows-latest.

macOS

Build sur macOS pour dmg / zip. Signature et notarisation Apple sont des étapes optionnelles supplémentaires (certificats, options mac dans electron-builder).

GitHub Actions (exemple de principe)

Trois jobs parallèles (ubuntu-latest, windows-latest, macos-latest) : checkout, npm ci, npm run icons, npm run dist, publication des artefacts du dossier release/.

Dépannage

  • Brouillon WordPress perdu : vérifier qu’IndexedDB n’est pas vidé (navigation privée, effacement des données du site). Tout effacer supprime volontairement le brouillon.
  • Mistral — clé invalide : message « Clé API Mistral invalide » ; régénérer une clé sur la console Mistral.
  • Mistral — limite : compléter les métadonnées à la main ; réessayer plus tard ou sur une image isolée via le bouton Mistral de la carte.
  • Zone de dépôt inactive (icônes) : vérifier que icon-script-protected.js est bien présent et chargé sans erreur dans la console.
  • Pack WordPress — scripts : chemins app/assets/js/wordpress/ référencés depuis app/pages/wordpress.html.
  • Page blanche après déplacement : utiliser app/pages/ ou npm start, pas d’anciens chemins à la racine (sauf redirections).
  • Icône Windows = logo Electron : exécuter npm run icons, vérifier build/icon.ico, puis reconstruire avec npm run dist sur Windows (ou CI Windows). Ne pas définir signAndEditExecutable: false (cela empêche d’injecter l’icône dans le .exe).
  • Icône manquante au build : npm run icons puis présence de build/icon.png et build/icon.ico.
  • Web bloqué après déploiement : domaine non listé dans AUTHORIZED_DOMAINS ou HTTPS requis selon license-protection.js.
  • Canvas / export vide : formats non supportés ou image corrompue ; tester avec un PNG simple.
  • CORS ou fetch config.env : en ouvrant les fichiers en file://, le fetch peut échouer ; les défauts du loader s’appliquent.
  • Cache Electron builder : répertoire utilisateur type ~/.cache/electron-builder.
  • Variable CI : avec CI=true, comportement non interactif.
